Architecture

One Apex call.
Zero external services.

A pipeline so simple it fits on one diagram. So fast it renders before your coffee finishes brewing.

1

Click the button

Drop the Crafter PDF LWC on any dashboard, report, or record page via Lightning App Builder.

→
2

Apex fetches data

One call to Reports.ReportManager.runReport() per dashboard component returns a serialized fact map.

→
3

Chart.js redraws

Each visualization is rendered onto an offscreen canvas in the user's browser.

→
4

jsPDF assembles

Vector PDF with your logo, brand colors, header, footer, page numbers, and cover page.

→
5

Save or send

Download locally, save to Files, push to ContentVersion, email, or fire from a Flow action.

Why this stack

The intersection nobody else hit.

Building this required four overlapping skills that almost never live in one team. We have all four.

⚡

Salesforce LWC + LWS fluency

Lightning Web Security blocks every screenshot-style approach. We build inside Lightning, not around it — using static resources, locker-safe libraries, and the Reports API the way Salesforce intended.

📐

jsPDF mastery

Font embedding, image scaling, page break math, vector tables — these are the unsexy details that make the difference between "ugly PDF export" and "a document your CFO would print and sign."

📊

Reports API depth

Fact maps, grouping, filter mutation, governor limits. Most senior Apex developers never touch this corner of the platform. We've shipped production code on it.

🧠

Anti-pattern discipline

Every previous attempt at this defaulted to Visualforce renderAs="pdf" (ugly) or external rendering (expensive). We had the discipline to say no to both and build what should exist.

Under the hood

Configured, not coded.

Admins build templates with Custom Metadata. Developers extend with Apex services. Both ship safely.

Custom Metadata template

Admins drop a logo, pick brand colors, choose sections — no code, no XML.

// CrafterPdfTemplate__mdt
Label: Q2 Sales Briefing
Brand_Color__c: #ff7a3d
Logo_Static_Resource__c: acme_logo
Page_Size__c: Letter
Cover_Page__c: true
Sections__c: [
  { type: 'dashboard', id: '01Z...' },
  { type: 'report',    id: '00O...' }
]

Flow action

Generate a PDF from any Flow — record-triggered, scheduled, or screen.

// In Flow Builder, drop the action:
Action: Generate Crafter PDF
Inputs:
  templateId     {!template.Id}
  recordId       {!Account.Id}
  deliverVia     'Email'
  recipientEmail {!owner.Email}
Outputs:
  contentVersionId
