Frequently asked

What people ask before buying.

How is this different from native Salesforce "Subscribe to Dashboard"?
Subscribe emails a static PNG image of one dashboard, on a fixed schedule, without branding. Crafter PDF generates branded, multi-section PDFs on-demand or on schedule, with your logo, colors, header, footer, cover page, and the ability to combine multiple dashboards and reports into a single document.
Why not use Conga or S-Docs?
Conga, Titan, and S-Docs are excellent at *transactional* documents (quotes, contracts, invoices). None of them render Salesforce dashboards natively — they all require you to template-build every chart manually using report data. Crafter PDF is built for dashboards from day one, costs 1/7th the price, and runs entirely client-side.
Does my data leave my Salesforce org?
No. The entire rendering pipeline runs in the user's browser, inside Salesforce. There are no callouts, no external rendering services, and no DPA required. This is the #1 reason enterprise security teams approve Crafter PDF in days instead of months.
Will it work with my existing Salesforce dashboards?
Yes — Crafter PDF reads dashboard component data via the standard Reports API, so any dashboard built on a standard Salesforce report works out of the box. CRM Analytics (Wave) dashboards are on the roadmap for v1.5.
What about Lightning Web Security?
Crafter PDF is built native to LWS — we don't try to screenshot the DOM (which LWS blocks). Instead, we re-render data from the Reports API using LWS-safe libraries shipped as static resources.
Is there a free trial?
Yes. The Free tier is permanently free for single-dashboard exports with default branding. The Team tier includes a 30-day free trial — no credit card required.
When will it be on the AppExchange?
We're targeting AppExchange security review submission in Q2 2026, with the listing live by Q3 2026. Until then, design partners can install via the unmanaged package directly from our GitHub repo.
Can I extend or customize it?
Absolutely. The Apex services and LWC components are designed to be extended. Custom Metadata-driven templates let admins build new layouts without code; our Apex APIs let developers integrate with any internal system.
